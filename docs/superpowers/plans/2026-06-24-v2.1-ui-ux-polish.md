# Elite Scope App v2.1 — UI/UX Polish — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the feedback-capture flow to review-before-send, remove the unused Proxy URL field, and improve readability/desktop layout — all front-end changes to `index.html`, no backend/transcription/AI-logic changes.

**Architecture:** All three tasks modify `index.html` only (no new files). The feedback redesign reuses the existing Settings-modal HTML/CSS pattern and the existing mic-button interaction pattern already used by the scope-line and notes mics. The Proxy URL removal deletes a dead code path. The readability/layout work is CSS-only plus a two-line JS fix for the desktop drag-and-drop ghost image.

**Tech Stack:** Vanilla JS, vanilla CSS — no new dependencies, no build step, consistent with the rest of this repo.

## Global Constraints

- No new build step, bundler, framework, or npm dependency.
- No changes to the JobTread integration logic, the AI review/flagging logic, the transcription pipeline internals, or the GitHub-issue-filing backend (`functions/*.js`, `lib/*.js`) — this round is `index.html` front-end only.
- Transcription latency work is explicitly out of scope for this round (deferred per the spec).
- The feedback button must never auto-send — every send requires an explicit user tap on a "Send" button.
- Readability sizing changes apply uniformly across phone and desktop, not as a separate "desktop mode."

```js
function onDragStart(e, index) {
  dragSrcIndex = index;
  e.dataTransfer.effectAllowed = 'move';
  e.dataTransfer.setData('text/plain', String(index));
  const items = document.querySelectorAll('.line-item');
  // Use the whole line as the drag ghost (not just the small drag handle,
  // which is the default since that's the element with draggable="true").
  if (items[index]) e.dataTransfer.setDragImage(items[index], 20, 20);
  // Delay adding the dimming class so the ghost image (captured above)
  // still looks like the normal, undimmed row.
  setTimeout(() => {
    if (items[index]) items[index].classList.add('dragging');
  }, 0);
}
```

Mobile touch dragging (`onTouchDragStart`/`onTouchDragMove`/`onTouchDragEnd`) uses a separate, custom-built implementation that doesn't generate a cursor-following ghost at all today — that's unchanged by this task, since the reported issue was specifically about desktop mouse dragging.
